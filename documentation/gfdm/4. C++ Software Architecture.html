4.1 Input Reader Module

Module Information
Associated header file /source_code/cpp/gfdm2d/input_readers/input_reader_module_linearHeatConduction.h
Associated cpp file /source_code/cpp/gfdm2d/input_readers/input_reader_module_linearHeatConduction.cpp
Libraries needed string, fstream, sstream, vector, Eigen/Core

1. Description

The Input Reader Module is the interface between the Input data generated in from the Preprocessor Object in Python and written into an input file and the C++ Solver routine. For this purpose the input file is read and the input data is stored in suitable containers.

2. Input Parameter

Signal Name Data-Type Known Size Comment
**_input.txt txt - input text file with all the necessary model information

3. Output Parameter

Signal Name Data-Type Known Size Comment
modelInput.simulationParameters.frequency double 1 Frequency of the simulation. Describes how many time steps are processed during one time unit
modelInput.simulationParameters.timeEnd double 1 The final time when the simulation stops
modelInput.simulationParameters.numThreads int 1 The number of threads used for multiphreading implementation
modelInput.simulationParameters.solverTolerance double 1 Tolerance of the solver
modelInput.simulationParameters.numNodes int 1 Number of nodes in the mesh
modelInput.simulationParameters.numTimesteps int 1 Number of timesteps
modelInput.simulationParameters.numBoundaries int 1 Number of defined boundaries in the mesh
modelInput.materialParameters.massDensity double 1 Specific Mass of the material
modelInput.materialParameters.specHeatCapacity double 1 specific heat capacity of the material
modelInput.materialParameters.thermalConductivity double 1 thermal conductiviry of the material
modelInput.initialConditions.startTemperature Array<double,Dynamic, 1> numNodes, 1 Array with the start temperatures for each node
modelInput.boundaryConditions.boundaryConditionPhysicalTag Array<int, Dynamic, 1> numBoundaries, 1 Array with the physical tag for each defined boundary
modelInput.boundaryConditions.boundaryConditionType Array<int, Dynamic, 1> numBoundaries, 1 Array with the boundary condition type for earch defined boundary:
1 = Dirichlet
1 = Neumann
modelInput.boundaryConditions.filmCoefficient Array<double,Dynamic, 1> numBoundaries, 1 Array with the film coefficients for each defined boundary
modelInput.boundaryConditions.heatflux Array<double, Dynamic, Dynamic> numBoundaries, numTimesteps Array with time dependent heatflux values for each defined boundary
modelInput.boundaryConditions.temperature Array<double, Dynamic, Dynamic> numBoundaries, numTimesteps Array with time dependent temperature values for each defined boundary
modelInput.meshFile string 1 Name of the msh file

4.3 Pointcloud Module (C++)

Module Information
Associated header file /source_code/cpp/gfdm2d/pointcloud_module.h
Associated cpp file /source_code/cpp/gfdm2d/pointcloud_module.cpp
Libraries needed meshmodule.h, Eigen/Core, iostream, algorithm, iterator, omp.h, math.h

1. Description

The Pointcloud Module used the information from the mesh to make the discretisation of the computational domain. This includes the calculation of normal vectors and the calculation of the support for each node.

2. Input Parameter

Signal Name Data-Type Known Size Comment
mesh.nodes.coord Array<double,Dynamic,3> numNodes, 3 Array with the coordinates of the nodes in 3 dimensions
mesh.boundaryElements.nodeTag Array<int,Dynamic,Dynamic> boundaryElements.numElements, 2 The index of the nodes included in the line element
mesh.innerElements.nodeTag Array<int,Dynamic,Dynamic> innerElements.numElements, 3 The index of the nodes for each triangle
N int 1 Global variable: number of nodes
numBoundNodes int 1 Global variable: number of boundary nodes, equal to boundaryElements.numElements

3. Output Parameter

Signal Name Data-Type Known Size Comment
pointcoud.coord Matrix<double, Dynamic, 2> numNodes, 2 Array with the coordinates of the nodes in 2 dimensions
pointcloud.boundaryTriangles Array<int,Dynamic,3> numBoundNodes, 3 Each line element on the boundary is connected to one triangle element. The indices of the corresponding nodes a saves
in this array
pointcloud.faceNormals Matrix<double, Dynamic, Dynamic> numBoundNodes, 2 Array with the face normals
pointcloud.nodeNormals Matrix<double, Dynamic, Dynamic> numBoundNodes, 2 Array with the node normals
pointcloud.support Array<vector<int>,Dynamic,1> N,1 Array from type vector which contains the support for each node
pointcloud.h double 1 support radius needed by the weight function

4. Flowchart

The support calculation is based on a neighbourhood search algorithm which uses the information from the array mesh.innerElements.nodeTag which stores the indices of the nodes included in each triangle. The Algorith looks as follows:

4.4 Boundary Condition Module (C++)

Module Information
Associated header file /source_code/cpp/gfdm2d/boundary_condition_module.h
Associated cpp file /source_code/cpp/gfdm2d/boundary_condition_module.cpp
Libraries needed Eigen/Core, mesh_module.h, input_reader_module_linearHeatConduction.h, omp.h

1. Description

The Boundary Conditions Module reads the defined boundary conditions and assembles them into the arrays for solving the global system of equations. It should specially consider the cases, when one particle is a member of two different boundaries and should define how to handle this.

2. Input Parameter

Signal Name Data-Type Known Size Comment
numBoundNodes int 1 Global variable: number of boundary nodes, equal to boundaryElements.numElements
N int 1 Global variable: number of nodes
modelInput.simulationParameters.numBoundaries int 1 Number of defined boundaries in the mesh
modelInput.boundaryConditions.boundaryConditionPhysicalTag Array<int, Dynamic,1> numBoundaries, 1 Array with the physical tag for each defined boundary
modelInput.boundaryConditions.boundaryConditionType Array<int, Dynamic, 1> numBoundaries, 1 Array with the boundary condition type for earch defined boundary:
1 = Dirichlet
2 = Neumann
mesh.boundaryElements.elementTag Array<int, Dynamic,1> boundaryElements.numElements, 1 The physical tag of each line element

3. Output Parameter

Signal Name Data-Type Known Size Comment
nodeTypes Array<unsigned int, Dynamic, 1> N, 1 Array with the node types:
0 = inner node
1 = dirichlet node
2 = neumann node
boundAllocMat Matrix<double, Dynamic, Dynamic> numBoundNodes, numBoundaries Global variable: boundary allocation matrix

4.6 Solver (main.cpp)

Module Information
Associated header file
Associated cpp file /source_code/cpp/linearHeatConduction2d/BiCGSTAB/main.cpp
Libraries needed string, fstream, Eigen/Core, Eigen/Sparse, omp.h, input_reader_module_linearHeatConduction.h, mesh_module.h, pointcloud_module.h, boundary_condition_module.h, output_writer_module.h, sparse_triplets_linearHeatConduction.h

1. Description

The Solvers solves the system of global equations for each defined time step and is directly written into the main.cpp file. The used solver algorithm is also defined there.

2. Input Parameter

Signal Name Data-Type Known Size Comment
name string 1 Project Name, first command line argument
workingDirectory string 1 Working directory, second command line argument
modelInput.simulationParameters.numThreads int 1 The number of threads used for multiphreading implementation
modelInput.simulationParameters.solverTolerance double 1 Tolerance of the solver
modelInput.simulationParameters.frequency double 1 Frequency of the simulation. Describes how many time steps are processed during one time unit
modelInput.simulationParameters.numTimesteps int 1 Number of timesteps
modelInput.simulationParameters.numBoundaries int 1 Number of defined boundaries in the mesh
mesh.nodes.numNodes int 1 Number of nodes
mesh.boundaryElements.numElements int 1 Number of boundary elements (lines)
modelInput.initialConditions.startTemperature Array<double,Dynamic, 1> numNodes, 1 Array with the start temperatures for each node
modelInput.boundaryConditions.temperature Array<double, Dynamic, Dynamic> numBoundaries, numTimesteps Array with time dependent temperature values for each defined boundary
modelInput.boundaryConditions.filmCoefficient Array<double,Dynamic, 1> numBoundaries, 1 Array with the film coefficients for each defined boundary
modelInput.boundaryConditions.heatflux Array<double, Dynamic, Dynamic> numBoundaries, numTimesteps Array with time dependent heatflux values for each defined boundary
B1 Array<double, Dynamic, 1> N, 1 Global variable: Right hand side of global system of equations: Saves coefficients from the heat equation
B2 Array<double, Dynamic, 1> numBoundNodes, 1 Global variable: Right hand side of global system of equations: Saves coefficients from the boundary conditions
boundAllocMat Matrix<double, Dynamic, Dynamic> numBoundNodes, numBoundaries Global variable: boundary allocation matrix

3. Output Parameter

Signal Name Data-Type Known Size Comment
solverOutput Matrix<double, Dynamic, 1> N, 1 Global variable: Array with the solver output. Changes each timestep
